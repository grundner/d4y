d4y

Git-native Runtime Platform für containerisierte Anwendungen — Sollzustand deklarativ, kontinuierlich selbstheilend.

Installation (1-Zeiler)

Auf einem Host mit Docker und öffentlichem DNS-A-Record:

D4Y_HOST=d4y.example.com D4Y_ACME_EMAIL=you@example.com \
  sh -c "$(curl -fsSL https://grundner.github.io/d4y/install.sh)"

Der Installer zieht das Image aus dem öffentlichen GHCR (anonym), erzeugt lokal ein Trigger-Token und einen Encryption-Key und startet d4y hinter Traefik mit automatischem TLS (ACME). d4y hält keine GitHub-Credentials.

Wie es funktioniert

  • Dein Config-Repo (privat) beschreibt den Sollzustand als YAML mit ${secret:NAME}-Platzhaltern.
  • Ein GitHub-Actions-Workflow pusht Sollzustand + Secrets an d4y (Vorlage).
  • d4y persistiert beides lokal (Secrets verschlüsselt) und reconciled kontinuierlich — auch ohne weiteren Push und über Neustarts hinweg.
